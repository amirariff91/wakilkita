# Impeccable Design Context — WakilKita

## Project
WakilKita is a dashboard-first civic nomination and issue intake for P105 Petaling Jaya. It lets residents submit local nominations, endorsements, profile claims, and issue priorities into a review queue. It is not online voting, not SPR-affiliated, not a party tool, and not official results.

## Emotional goal
Serious, plain, accountable. It should feel like a civil-society intake desk with an operator dashboard, not a startup pitch, glossy election dashboard, or AI-generated brand concept.

## Better-wow direction: P105 Civic Intake Desk

Target: a precise civil-society intake desk — local to Petaling Jaya, plain but intentionally designed, privacy-first, dashboard-first, visibly non-official, calm enough for trust-sensitive civic use.

Wow comes from craft and confidence: a restrained WakilKita mark, a subtle P105/PJ line motif, a clearer resident journey, a better intake form state system, a review dashboard with careful status language, sharper trust boundaries, and less default-box UI.

### Allowed visual techniques
- Inline SVG or static SVG files only.
- CSS transitions and keyframe animations only — no Lottie, Framer Motion, GSAP, or Rive.
- 1px borders, deliberate dividers, and very light shadows.
- Off-white / white / ink / blue-slate / cool grey / burnt orange from the defined palette.
- Motion is short, non-looping, and never required to understand content.

### Banned visual techniques
- Three.js, WebGL, map SDKs, canvas-based graphics, particles, or any 3D rendering.
- Fake map data, fake constituency heatmaps, fake counts, fake public leaderboards.
- Person/representative profile cards with generated or placeholder avatars.
- Any visual that could be mistaken for official election infrastructure.
- Looping animations or motion that is required to understand content.
- Ballot imagery, crests, shields, scales, parliament shapes, flags, stars, laurels, seals, or official badges.

### Acceptance criteria for first screen
1. P105 Petaling Jaya is identified before any form input.
2. Non-voting boundary is visible before or at the form — not only in the footer.
3. The intake form and dashboard route are primary actions, above or alongside explanatory content.
4. No IC/eKYC collection message is visible before the submit button.
5. No visual suggests voting, official status, public counts, or live results.
6. No email handoff for nominations, endorsements, or issue intake.

## Page direction
- Petaling Jaya is the lead constituency. Do not reintroduce Pandan-first framing or Pandan data on the public page.
- Core intake must appear before any explanatory content.
- Public copy should lead with current operational truth: dashboard queue, manual review, IC/eKYC separated from public form data, no public supporter lists, consent before public profiles.
- Do not foreground future verification, public counts, support totals, architecture diagrams, or profile-state mockups until those systems and policies exist.
- Do not use founder-facing terms like “MVP”, “prototype”, “pilot”, “preview”, “demo”, or “product review” in public copy.
- Do not use “secure” unless the implementation provides and explains a real security guarantee.

## Dashboard direction
- Dashboard is a review queue, not a public leaderboard.
- Status language should be operational: submitted, needs review, duplicate check, consent needed, approved, declined.
- Hide private reply contact in list views. Show “stored privately” only.
- Flag possible IC, phone, address, or allegation text for reviewer caution.
- Avoid ranking, totals-by-person, turnout, vote, mandate, or candidate winner language.

## Visual system
- Palette: off-white #F8F8F6, white cards #FFFFFF, ink #111111, blue-slate #263A4F, dark slate #0F1724, cool grey #E8ECF0 / #F0F2F3, burnt orange #B35C1E, body slate #4D5650.
- Typography: sans-first. Use Public Sans / Source Sans 3 / system sans stack. Avoid Georgia or dramatic serif landing-page headings.
- Shape: flatter civic utility UI. Prefer square or small-radius panels, 1px borders, light shadows only.
- Motion: minimal; trust-sensitive UI should not feel gimmicky.

## Anti-patterns to avoid
- No generic startup hero with vague gradients.
- No warm parchment + mint civic-tech default treatment.
- No crypto/blockchain theatre.
- No public-facing claims of official civic authority.
- No ballot imagery, party/campaign styling, government seals, or fake live counts.
- No decorative dashboards, generated evidence scenes, or placeholder representative cards.
- No individual political preference exposure.
- No thick left-accent civic cards, nested card piles, or template feature grids.
